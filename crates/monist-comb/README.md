# `monist-comb`

**The Interaction Net Backend (WebGPU)**

`monist-comb` is the physical execution hardware layer of the Monist Engine. It operates under a "destroy the variable" compiler paradigm. Taking the strictly verified topological bounds from the CPU layer, it replaces classical lexical scope with pure, untyped combinatory logic ($S, K, I$) mapped to De Bruijn indices.

By burning the typestate limits generated by `monist-core` directly into the structural graph via $T$-functor wrappers, `monist-comb` bypasses traditional garbage collection. It dispatches the graph into VRAM, executing logic entirely in parallel utilizing lock-free Compare-And-Swap (CAS) atomic operations within native WGSL compute shaders.
