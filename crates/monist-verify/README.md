# `monist-verify`

**The Validation Gateway**

`monist-verify` connects the bare-metal evaluations of the Monist Engine to absolute formal truth. It takes the bounded physical topologies generated by the CPU and translates them into an SMT-LIB trace (`NfValidate.StratificationWitness`).

This trace is entirely language-agnostic and acts as the explicit bridge to the **NF-Sketches** formal lab built in Lean 4. The `parse-strat` interpreter ingests this SMT-LIB witness to formally verify cross-language geometric equivalence against proven theorems like `StratificationSoundness.lean`.

## Architecture Pipeline

1. [`monist-core`](../monist-core/README.md) & [`monist-psg`](../monist-psg/README.md): Provide the fully bounded, paradox-free physical topology matrix.
2. **`monist-verify`** 📍 *You are here*
3. **NF-Sketches (Lean 4)**: The generated `.smt` output is piped externally to `lake exe parse-strat` for validation.
4. [`monist-comb`](../monist-comb/README.md): Once verified, the exact same directed graph is dispatched to the GPU for execution.
